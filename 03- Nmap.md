# Layer 1
- **Nmap = Network Mapper**
- مكتوبة بلغات زي ال CPP / Python 
- - وظيفتها الأساسية:
    
    1. **عمل فحص (Scan) للشبكات** لمعرفة الأجهزة (Hosts) اللي شغالة.
        
    2. **معرفة البورتات والخدمات اللي بتشتغل** على كل جهاز (مع الاسم والنسخة لو ممكن).
        
    3. **تحديد نظام التشغيل (OS Detection)** وإصداره.
        
    4. **تحليل إعدادات الأمان** زي الـ Firewall والـ IDS (Intrusion Detection System).

### **أهم استخدامات Nmap**

- **تدقيق أمان الشبكات (Security Audit).**
    
- **محاكاة اختبارات الاختراق (Penetration Testing).**
    
- **فحص إعدادات الـ Firewall والـ IDS.**
    
- **رسم خريطة الشبكة (Network Mapping).**
    
- **تحليل الاستجابات (Response Analysis).**
    
- **تحديد البورتات المفتوحة (Open Ports).**
    
- **تقييم الثغرات (Vulnerability Assessment).**

### **هيكل Nmap – Nmap Architecture**

تقريبًا أي فحص باستخدام Nmap بيتقسم للمراحل دي:

1. **Host Discovery** → هل الجهاز شغال ولا لا؟
    
2. **Port Scanning** → أي بورتات مفتوحة؟
    
3. **Service Enumeration** → إيه الخدمات اللي شغالة على البورتات دي؟
    
4. **OS Detection** → نظام التشغيل وإصداره.
    
5. **Nmap Scripting Engine (NSE)** → تعمل سكريبتات جاهزة للفحص المتقدم (زي البحث عن ثغرات أو كلمات سر ضعيفة).
    

---

### **الصيغة العامة للأوامر**

`nmap <scan types> <options> <target>`

- `<scan types>` = نوع الفحص (مثلاً -sS أو -sU).
    
- `<options>` = خيارات إضافية (زي -p للبورتات أو -A للفحص الكامل).
    
- `<target>` = الهدف (IP أو اسم دومين).

### **أنواع الـ Scans في Nmap**

Nmap بيقدم أنواع كتيرة جدًا من الـ Scans، كل نوع له طريقة مختلفة في إرسال الـ Packets:

#### **1. TCP Scans**

- `-sS` → **SYN Scan** (الأكثر استخدامًا والأسرع).
    
    - بيبعت SYN Packet.
        
    - لو السيرفر رد بـ **SYN-ACK** → البورت **Open**.
        
    - لو رد بـ **RST** → البورت **Closed**.
        
    - لو مفيش رد → البورت **Filtered** (Firewall بيمنع الرد).
        
    - الميزة: الفحص سريع ومش بيكمل الـ TCP Handshake (Stealth Scan).
![[Pasted image 20250825100410.png]]


- `-sT` → **Connect Scan**.
    
    - بيعمل الـ 3-way Handshake كامل (أبطأ – أسهل في الاكتشاف).
        
- `-sA` → **ACK Scan**.
    
    - بيتأكد هل البورت محمي بـ Firewall ولا لا (مش بيقولك Open أو Closed).
        
- `-sW` → **Window Scan** (نادر الاستخدام – بيعتمد على Window size في الردود).
    
- `-sM` → **Maimon Scan** (اختبار استجابات معينة).
    

---

#### **2. UDP Scan**

- `-sU` → **UDP Scan**
    
    - أصعب من الـ TCP لأنه بدون Handshake.
        
    - لو السيرفر بيرد → البورت Open.
        
    - لو بيبعت ICMP Unreachable → البورت Closed.
        
    - لو مفيش رد → ممكن يكون Open أو Firewall بيمنع.

# Layer 2
## **ليه بنعمل Host Discovery قبل أي حاجة؟**

- في اختبار الاختراق الداخلي (Internal Penetration Test) لازم نعرف **أي الأجهزة شغّالة (Online)** جوة الشبكة قبل ما نبدأ نفحص البورتات أو الخدمات.
    
- **Nmap بيوفّر طرق متعددة** علشان يتأكد إذا الهدف (Host) حي ولا ميت (Up or Down).
    
- أهم طريقة وأسهلها: **ICMP Echo Requests (Ping)** → لو الجهاز بيرد يبقى شغال.
    
- لكن لو فيه Firewall بيمنع الـ Ping، نحتاج لطرق تانية (هنشوفها لاحقًا في Firewall & IDS Evasion).
    

---

## **ملاحظات مهمة:**

1. **لازم تخزن كل Scan** (باستخدام `-oA`) علشان تقدر ترجع له وتقارن النتائج أو تستخدمها في التقرير.
    
2. **أدوات مختلفة ممكن تطلع نتائج مختلفة**، فتوثيق النتائج مهم علشان تعرف الفرق بينهم.
    

---

## **السيناريوهات المختلفة لفحص الأجهزة (Host Discovery)**

---

### **1. فحص شبكة كاملة (Network Range Scan)**

لو عايز تشوف كل الأجهزة اللي شغالة في رينج كامل (مثلاً /24):

`sudo nmap 10.129.2.0/24 -sn -oA tnet | grep for | cut -d" " -f5`

**شرح الأوامر:**

- `10.129.2.0/24` → نطاق الشبكة.
    
- `-sn` → تعطيل فحص البورتات، نكتفي بفحص الأجهزة الحية.
    
- `-oA tnet` → حفظ النتائج بكل الصيغ (nmap, xml, grepable) باسم tnet.
    
- `grep for | cut -d" " -f5` → يعرض فقط الـ IPs اللي شغالة.
    

**النتيجة:**

`10.129.2.4 10.129.2.10 10.129.2.11 10.129.2.18 10.129.2.19 10.129.2.20 10.129.2.28`

> ملاحظة: لو الـ Firewall بيمنع الـ Ping، الأجهزة ممكن تظهر كأنها Down رغم إنها شغالة.

---

### **2. فحص قائمة IP جاهزة (Scan IP List)**

لو معاك ملف hosts.lst فيه الـ IPs:

`cat hosts.lst`

**المحتوى:**

`10.129.2.4 10.129.2.10 10.129.2.11 10.129.2.18 10.129.2.19 10.129.2.20 10.129.2.28`

**تشغيل الفحص:**

`sudo nmap -sn -oA tnet -iL hosts.lst | grep for | cut -d" " -f5`

- `-iL hosts.lst` → الفحص يتم على كل IP موجود في الملف.
    

**النتيجة:**

`10.129.2.18 10.129.2.19 10.129.2.20`

> يعني بس 3 أجهزة من 7 ردوا. الباقي ممكن يكون واصل لكن بيمنع الـ ICMP Echo.

---

### **3. فحص مجموعة IP صغيرة (Scan Multiple IPs)**

لو عندك IPs متفرقة:

`sudo nmap -sn -oA tnet 10.129.2.18 10.129.2.19 10.129.2.20 | grep for | cut -d" " -f5`

**أو كرينج متتالي:**

`sudo nmap -sn -oA tnet 10.129.2.18-20 | grep for | cut -d" " -f5`

**النتيجة:**

`10.129.2.18 10.129.2.19 10.129.2.20`

---

### **4. فحص IP واحد (Single Host Discovery)**

قبل ما تفحص بورتات وخدمات، تأكد إن الجهاز حي:

`sudo nmap 10.129.2.18 -sn -oA host`

**الناتج:**

`Nmap scan report for 10.129.2.18 Host is up (0.087s latency). MAC Address: DE:AD:00:00:BE:EF`

- `-sn` → تعطيل فحص البورتات.
    
- `-oA host` → حفظ النتيجة باسم host.
    

---

## **تفاصيل تقنية مهمة عن الـ Ping في Nmap**

- بشكل افتراضي: Nmap بيستخدم **ARP Ping أولًا** (في الشبكات المحلية LAN).
    
- بعد كده ممكن يرسل **ICMP Echo Request (-PE)** لو عايز تجبره.
    
- استخدم `--packet-trace` علشان تشوف كل الـ Packets المرسلة والمستقبَلة.
    
- استخدم `--reason` علشان تعرف **ليه Nmap صنف الجهاز إنه Up** (بناءً على ARP أو ICMP).
    
- لو مش عايز يستخدم ARP نهائيًا: استخدم `--disable-arp-ping`.
    

---

### **مثال: فرض استخدام ICMP Echo بدل ARP**

`sudo nmap 10.129.2.18 -sn -oA host -PE --packet-trace`

**الناتج:**

`SENT ARP who-has 10.129.2.18 RCVD ARP reply 10.129.2.18 is-at DE:AD:00:00:BE:EF Host is up (0.023s latency). MAC Address: DE:AD:00:00:BE:EF`

- هنا Nmap اعتبر الجهاز Up من خلال رد الـ ARP.
    

---

### **تعطيل الـ ARP Ping وإرسال ICMP فقط**

`sudo nmap 10.129.2.18 -sn -oA host -PE --packet-trace --disable-arp-ping`

**الناتج:**

`SENT ICMP Echo request RCVD ICMP Echo reply Host is up (0.086s latency).`

- هنا أجبرنا Nmap يستخدم ICMP فقط بدون ARP.
    

---

## **الخلاصة**

1. **ابدأ دائمًا بـ Host Discovery قبل أي Port Scan.**
    
2. **استخدم -sn لتعطيل فحص البورتات واكتشاف الأجهزة فقط.**
    
3. **احفظ كل النتائج (-oA) للمقارنة والتوثيق.**
    
4. **ARP Ping أسرع في LAN، وICMP مفيد لو عايز فحص كلاسيكي.**
    
5. **--reason و --packet-trace أدوات مهمة لفهم النتائج.**

# Layer 3
شرح هذا الجزء يتناول **مسح المنافذ (Port Scanning)** باستخدام أداة **Nmap**، وكيفية تحليل النتائج بدقة لفهم حالة كل منفذ والخدمات التي تعمل عليه. سأقسم لك الشرح إلى نقاط رئيسية وبسيطة:

---

## **1. الهدف من Host و Port Scanning**

بعد أن نتحقق أن الهدف على الشبكة (Host is up)، نريد جمع معلومات أعمق عن النظام، مثل:

- المنافذ المفتوحة والخدمات عليها.
    
- إصدارات الخدمات (Service Versions).
    
- نوع نظام التشغيل.
    
- أي تفاصيل أخرى يمكن استخراجها من هذه الخدمات.
    

---

## **2. حالات المنافذ (6 States of Ports)**

أي منفذ تفحصه Nmap سيكون في حالة من الحالات التالية:

| الحالة            | الشرح                                                                                                                                                           |
| ----------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **open**          | المنفذ مفتوح ويستقبل اتصالات.                                                                                                                                   |
| **closed**        | المنفذ مغلق والجهاز يرد بحزم TCP تحتوي `RST`.                                                                                                                   |
| **filtered**      | لا رد أو رد غامض (Firewall يمنع تحديد الحالة).                                                                                                                  |
| **unfiltered**    | المنفذ متاح للوصول لكن غير معروف إذا كان مفتوح أو مغلق (يظهر في TCP-ACK scan).                                                                                  |
| open \| filterd   | \|If we do not get a response for a specific port, `Nmap` will set it to that state. This indicates that a firewall or packet filter may protect the port.\|    |
| closed \| filterd | This state only occurs in the **IP ID idle** scans and indicates that it was impossible to determine if the scanned port is closed or filtered by a firewall.\| |

---

## **3. مسح منافذ TCP (SYN Scan vs Connect Scan)**

### **أولاً: SYN Scan (-sS)**

- هو الفحص الافتراضي عند تشغيل Nmap كـ root.
    
- يسمى **Half-open scan** لأنه يرسل SYN، ينتظر SYN-ACK، ثم يغلق الاتصال قبل اكتماله.
    
- سريع وأكثر **تخفيًا** (لا يكمل الـ TCP handshake بالكامل).
    

### **ثانياً: Connect Scan (-sT)**

- يتم إذا لم يكن لديك صلاحيات root.
    
- يكمل الـ **TCP 3-Way Handshake** (SYN → SYN-ACK → ACK).
    
- **أكثر دقة** لكن **أقل تخفيًا** (يُسجل في سجلات السيرفر).
    
- جيد إذا أردت التأكد بدقة أو إذا كان جدار الحماية يسمح بالاتصالات الصادرة فقط.
    

---

### **مسح أفضل 10 منافذ شائعة**

```bash
sudo nmap 10.129.2.28 --top-ports=10
```

- يظهر المنافذ وحالتها (open, closed, filtered).
    
- مثال: منفذ 22 مفتوح (SSH)، منفذ 445 محجوب (filtered).
    
- إيه هما التوب 10 ؟

### **استخدام --packet-trace لمعرفة ما يحدث بالضبط**

```bash
sudo nmap 10.129.2.28 -p 21 --packet-trace -Pn -n --disable-arp-ping
```

- يوضح الحزم المرسلة (SYN) والردود (RST-ACK).
    
- **SENT** تعني حزمة أرسلت.
    
- **RCVD** تعني حزمة استُقبلت.
    - -Pn -> بتقول متعملش بينج قبل الفحص يعني بفترض ان الجهاز اصلًا شغال
    - - n -> بتلغي ال DNS

### **Connect Scan على منفذ HTTPS (443)**

```bash
sudo nmap 10.129.2.28 -p 443 --packet-trace --disable-arp-ping -Pn -n -sT --reason
```

- إذا جاء رد SYN-ACK → المنفذ مفتوح.
    
- إذا جاء RST → المنفذ مغلق.
    
The `Connect` scan (also known as a full TCP connect scan)

## **5. حالة Filtered Ports (جدار الحماية)**

- إذا كان الـ firewall **يسقط الحزم** → لا رد، Nmap يعيد المحاولة 10 مرات (بطيء).
    
- إذا كان الـ firewall **يرفض الحزم** → يرد بـ ICMP (type 3/code 3 = port unreachable).
    

أمثلة:

- منفذ 139 → لا رد → filtered.
    ![[Pasted image 20250826101829.png]]
    
- منفذ 445 → رد ICMP unreachable → filtered بشكل واضح.
    
![[Pasted image 20250826101938.png]]

---

## **6. مسح منافذ UDP (-sU)**

- أبطأ من TCP لأنه **بلا handshake** (stateless).
    
- غالباً لا يوجد رد إذا كان المنفذ مفتوح.
    
- إذا المنفذ مغلق → الجهاز يرسل ICMP port unreachable.
    
- إذا الرد غامض → الحالة open|filtered.
    

أمثلة:

```bash
sudo nmap 10.129.2.28 -F -sU       # مسح سريع لأفضل 100 منفذ UDP
sudo nmap 10.129.2.28 -sU -p 137 --packet-trace --reason
sudo nmap 10.129.2.28 -sU -p 100 --packet-trace --reason
```

---

## **7. تحديد إصدار الخدمة (-sV)**

يمكنك معرفة إصدار الخدمة ونوعها:

```bash
sudo nmap 10.129.2.28 -p 445 -sV --reason -Pn -n --disable-arp-ping --packet-trace
```

- مفيد جداً لاختيار الـ exploit المناسب.
    
- يظهر تفاصيل مثل: نوع SMB أو Apache أو SSH وإصداره.
    
![[Pasted image 20250826102235.png]]

### **الخلاصة**

- **SYN Scan (-sS)** سريع وتخفي.
    
- **Connect Scan (-sT)** أدق لكن مكشوف.
    
- **UDP Scan (-sU)** أبطأ وأصعب في التفسير.
    
- **Filtered ports** تعني أن firewall موجود.
    
- **استخدم -sV لمعرفة الإصدارات** لتخطيط الهجمات.
    
- **استخدم --packet-trace لفهم حركة الحزم** أثناء الفحص.

## **١. إحنا ليه بنعمل Host و Port Scanning؟**

بعد ما نتاكد إن الجهاز الهدف موجود على الشبكة (Host Up)، بنبدأ ندور على معلومات أكتر زى:

- إيه المنافذ (Ports) اللي مفتوحة عليه.
    
- الخدمات (Services) اللي شغالة على كل منفذ.
    
- الإصدارات بتاعة الخدمات دي.
    
- محاولة معرفة نوع نظام التشغيل (OS).  
    الكلام ده كله مهم عشان تعرف إنت ممكن تهاجم إزاي.
    

---

## **٢. حالات المنافذ (Port States)**

أي منفذ ممكن يكون في واحدة من ٦ حالات:

1. **Open (مفتوح):** الخدمة شغالة والمنفذ بيرد على أي محاولة اتصال.
    
2. **Closed (مقفول):** الجهاز بيرد بحزمة TCP فيها RST → يعني "أنا هنا بس مفيش خدمة على المنفذ".
    
3. **Filtered (متفلتر):** مفيش رد واضح (Firewall بيمنعك تعرف المنفذ مفتوح ولا مقفول).
    
4. **Unfiltered (مش متفلتر):** المنفذ متاح، بس مش معروف إذا كان مفتوح أو لأ (بيظهر في TCP-ACK Scan).
    
5. **Open|Filtered (مفتوح أو متفلتر):** Nmap مش قادر يحدد لأن الرد مش واضح أو مش موجود.
    
6. **Closed|Filtered (مقفول أو متفلتر):** حالة خاصة بتحصل مع IP ID idle scan لما مينفعش تحدد.
    

---

## **٣. أنواع مسح TCP (SYN Scan vs Connect Scan)**

### **SYN Scan (-sS):**

- ده الافتراضي لو بتشغل Nmap كـ root.
    
- بيبعت SYN، لو السيرفر رد بـ SYN-ACK يبقى المنفذ مفتوح → بس قبل ما يكمل الـ Handshake بيقف.
    
- بيسموه Half-open scan → سريع وأكتر تخفي.
    

### **Connect Scan (-sT):**

- بيتعمل لو مش شغال كـ root.
    
- بيكمل الـ TCP Handshake بالكامل (SYN → SYN-ACK → ACK).
    
- أدق لكن مكشوف أكتر (هتلاقيه في الـ Logs).
    
- كويس لما تحب تتأكد بنسبة ١٠٠٪ إن المنفذ مفتوح.
    

---

## **٤. أمثلة على أوامر Nmap**

### **مسح أفضل ١٠ منافذ شائعة:**

```bash
sudo nmap 10.129.2.28 --top-ports=10
```

- بيطلعلك أكتر المنافذ المشهورة وحالتها.
    
- مثال: منفذ 22 مفتوح (SSH)، منفذ 445 متفلتر (Firewall).
    

---

### **استخدام --packet-trace لمتابعة الحزم:**

```bash
sudo nmap 10.129.2.28 -p 21 --packet-trace -Pn -n --disable-arp-ping
```

- **SENT:** الحزم اللي انت بعتها.
    
- **RCVD:** الحزم اللي رجعتلك.
    
- تقدر تعرف الـ RST يعني المنفذ مقفول.
    

---

### **Connect Scan على منفذ 443:**

```bash
sudo nmap 10.129.2.28 -p 443 --packet-trace --disable-arp-ping -Pn -n -sT --reason
```

- لو رد بـ SYN-ACK → مفتوح.
    
- لو رد بـ RST → مقفول.
    

---

## **٥. المنافذ المتفلترة (Filtered Ports)**

- **Firewall بيعمل Drop:** مش بيرد خالص → Nmap يعيد المحاولة كذا مرة (بطئ).
    
- **Firewall بيعمل Reject:** بيرد بـ ICMP Type 3/Code 3 → "Port unreachable".
    

أمثلة:

- منفذ 139 → مفيش رد → Filtered.
    
- منفذ 445 → بيرد ICMP unreachable → واضح إنه محجوب.
    

---

## **٦. مسح منافذ UDP (-sU)**

- أبطأ من TCP لأنه مفيش Handshake.
    
- المنفذ المفتوح غالباً مش هيرد.
    
- المنفذ المقفول → الجهاز بيرد ICMP Port Unreachable.
    
- المنفذ الغامض → بيظهر Open|Filtered.
    

أوامر:

```bash
sudo nmap 10.129.2.28 -F -sU       # مسح سريع لأفضل 100 منفذ UDP
sudo nmap 10.129.2.28 -sU -p 137 --packet-trace --reason
sudo nmap 10.129.2.28 -sU -p 100 --packet-trace --reason
```

---

## **٧. معرفة إصدار الخدمة (-sV)**

عشان تعرف نوع وإصدار السيرفر والخدمة:

```bash
sudo nmap 10.129.2.28 -p 445 -sV --reason -Pn -n --disable-arp-ping --packet-trace
```

- ممكن يطلعلك مثلاً: SMB v2.0 أو Apache 2.4.41 أو SSH 7.9.
    
- المعلومة دي مهمة جداً لاختيار الـ Exploit المناسب.
    

---

## **الخلاصة:**

- **SYN Scan** سريع ومتخفي.
    
- **Connect Scan** أدق لكن بيتسجل.
    
- **UDP Scan** بطئ وصعب شوية في التحليل.
    
- **Filtered = Firewall موجود.**
    
- **-sV عشان تعرف الإصدار وتستهدفه.**

# Layer 6
## **إيه هو الـ Nmap Scripting Engine (NSE)؟**

الـ **NSE** ميزة قوية في Nmap بتخليك تشغّل سكريبتات مكتوبة بلغة **Lua** عشان:

- تكشف معلومات زيادة عن السيرفيس.
    
- تعمل فحص ثغرات أو Brute Force أو حتى Exploit.
    
- تقدر كمان تكتب سكريبت بنفسك لو عايز.
    

السكريبتات دي متقسمة إلى **14 تصنيف**:

1. **auth** → بتجرب تكتشف الـ Credentials (اسم المستخدم والباسورد).
    
2. **broadcast** → بتعمل اكتشاف أجهزة بالشبكة عن طريق البث (Broadcast).
    
3. **brute** → بتجرب Brute Force على الخدمات.
    
4. **default** → السكريبتات الافتراضية اللي بتشتغل مع `-sC`.
    
5. **discovery** → بتساعدك تكتشف الخدمات المتاحة.
    
6. **dos** → بتختبر إذا السيرفيس ممكن يحصلها DoS (مش بنستخدمها كتير لأنها ممكن تبوظ السيرفيس).
    
7. **exploit** → بتحاول تستغل ثغرة موجودة أصلاً في الخدمة.
    
8. **external** → سكريبتات بتستخدم خدمات بره الجهاز للتحليل.
    
9. **fuzzer** → بتبعت Requests مختلفة وغريبة عشان تكشف Bugs أو ثغرات.
    
10. **intrusive** → فحوصات ممكن تأثر على النظام المستهدف أو تبطّله.
    
11. **malware** → بتكشف إذا فيه Malware مصاب بيه السيرفر.
    
12. **safe** → فحوصات آمنة مش هتبوظ حاجة.
    
13. **version** → بتحسن كشف إصدارات الخدمات.
    
14. **vuln** → بتكشف الثغرات المعروفة وتربطها بقاعدة بيانات الـ CVE.
    

---

## **إزاي تشغل سكريبتات الـ NSE؟**

### 1. **تشغيل السكريبتات الافتراضية**:

`sudo nmap <target> -sC`

- هنا بيشغّل كل سكريبتات الـ Default Category.
    

---

### 2. **تشغيل تصنيف كامل من السكريبتات**:

`sudo nmap <target> --script <category>`

مثلاً:

`sudo nmap 10.129.2.28 --script vuln`

- هيشغّل كل سكريبتات كشف الثغرات على التارجت.
    

---

### 3. **تشغيل سكريبتات معينة بالاسم**:

`sudo nmap <target> --script <script-name>,<script-name>`

مثلاً:

`sudo nmap 10.129.2.28 -p 25 --script banner,smtp-commands`

- سكريبت `banner` → بيطبع Banner الخدمة (ممكن تعرف النظام أو التطبيق من أول Response).
    
- سكريبت `smtp-commands` → بيكشف أوامر SMTP المتاحة (ممكن يساعدك تتعرف على اليوزرز).

# Layer 7
هذا الجزء يشرح **كيفية تحسين سرعة أداء عمليات الفحص في Nmap**، لكنه يحذرك أيضاً من المخاطر المصاحبة لتسريع الفحص أكثر من اللازم. الفكرة الأساسية: كلما جعلت الفحص أسرع، زاد احتمال فقدان نتائج أو إثارة تنبيه لدى أنظمة الحماية.

---

## **1. ما معنى Performance في Nmap؟**

Nmap عندما يفحص شبكة، يرسل حزم (packets) وينتظر الرد (RTT = Round Trip Time). لو كانت الشبكة كبيرة أو الاتصال بطيء، الفحص قد يأخذ وقت طويل. لذلك يوفر Nmap خيارات لضبط السرعة والدقة:

- **-T 0-5** : قوالب جاهزة للتحكم في سرعة الفحص.
    
- **--min-rate / --max-retries / --initial-rtt-timeout** : إعدادات دقيقة للتحكم في الحزم والوقت.
    

---

## **2. Timeouts (أوقات الانتظار)**

- Nmap يبدأ عادة بـ **timeout = 100ms** للرد.
    
- يمكنك تقليله (مثلاً 50ms) لتسريع الفحص.
    
- **الخطر:** إذا كان الهدف أبطأ من المتوقع، قد تعتبره غير موجود (false negative).  
    **مثال:**
    

`sudo nmap 10.129.2.0/24 -F                  # timeout الافتراضي → 39 ثانية sudo nmap 10.129.2.0/24 -F --initial-rtt-timeout 50ms --max-rtt-timeout 100ms   # أسرع → 12 ثانية لكن أقل نتائج`

---

## **3. Max Retries (عدد مرات إعادة الإرسال)**

- Nmap يعيد المحاولة **10 مرات افتراضياً** لكل منفذ.
    
- تقليلها لـ 0 يجعل الفحص أسرع بكثير، لكن قد تفقد بعض المنافذ المفتوحة إذا لم ترد من أول مرة.  
    **مثال:**
    

`sudo nmap 10.129.2.0/24 -F | grep "/tcp" | wc -l      # 23 منفذ مفتوح sudo nmap 10.129.2.0/24 -F --max-retries 0 | grep "/tcp" | wc -l   # 21 منفذ فقط`

---

## **4. Rates (عدد الحزم في الثانية)**

- **--min-rate 300** يعني إرسال 300 حزمة في الثانية على الأقل.
    
- مفيد في اختبار اختراق أبيض (white-box) حيث يسمح لك بتجاوز أنظمة الحماية.  
    **مثال:**
    

`sudo nmap 10.129.2.0/24 -F --min-rate 300`

سرعة الفحص زادت من 30 ثانية إلى 8 ثوانٍ **بدون فقد منافذ**.

---

## **5. Timing Templates (-T)**

- **-T 0 Paranoid** → بطيء جداً، مناسب لتجنب اكتشاف IDS.
    
- **-T 1 Sneaky** → بطيء ولكن أسرع من 0.
    
- **-T 2 Polite** → أبطأ من الوضع الافتراضي (يقلل استهلاك الشبكة).
    
- **-T 3 Normal** → الوضع الافتراضي.
    
- **-T 4 Aggressive** → سريع، قد يثير تنبيهات الحماية.
    
- **-T 5 Insane** → أسرع شيء، خطير جداً في بيئات حقيقية لأنه يولد ضجيج كبير.
    

**مثال:**

`sudo nmap 10.129.2.0/24 -F -T 5`

نفس النتائج لكن وقت أقل (32 ثانية → 18 ثانية).

---

## **الخلاصة**

- إذا تريد السرعة **بأي ثمن** → قلل الـ RTT و الـ retries أو استخدم **-T 4 أو -T 5**.
    
- إذا تريد **دقة عالية** → اترك الإعدادات الافتراضية أو استخدم **-T 3**.
    
- استخدم **--min-rate** لو عندك إنترنت سريع ومسموح لك تعمل فحص قوي.
    
- لا تسرع أكثر من اللازم في بيئة Black-box (غير مصرح لك)، لأنك ممكن تُكتشف أو تفقد نتائج مهمة.

# Layer 8
هذا القسم يشرح **كيف تتجاوز جدران الحماية (firewalls) وأنظمة كشف/منع التسلل (IDS/IPS) باستخدام Nmap**. الهدف هو التعرف على الحماية الموجودة لدى الهدف، ثم إجراء الفحص بشكل خفي أو تخطي القواعد التي تمنع الوصول.

---

## **1. ما الفرق بين Firewall و IDS/IPS؟**

- **Firewall**: يراقب الحزم ويقرر السماح أو الحظر بناءً على قواعد (Port، IP، Protocol). يمكن أن يسقط الحزم (drop) أو يرفضها بإرسال رسالة (reject).
> Firewall -> Against unauthorized connection attempts from external networks
- **IDS (Intrusion Detection System)**: يراقب الشبكة ويكتشف الهجمات المحتملة ويبلغ المسؤول.
    > scans the network for potential attacks, analyzes them, and reports any detected attacks
- **IPS (Intrusion Prevention System)**: مثل IDS لكن يتخذ إجراءات تلقائية (حظر IP، إغلاق اتصال...).
    > complements `IDS` by taking specific defensive measures if a potential attack should have been detected

---

## **2. كيف تعرف أن هناك Firewall؟**

- عندما يظهر المنفذ **filtered** بدل open/closed → يعني الحزم تم إسقاطها بدون رد (drop) أو رفضها برسالة ICMP/RST (reject).
    
- الأخطاء المحتملة:
    
    - Net Unreachable / Prohibited
        
    - Host Unreachable / Prohibited
        
    - Port Unreachable
        
    - Proto Unreachable
        

---

## **3. أنواع الفحص لتجاوز الجدار الناري**

### **SYN Scan (-sS)**

يرسل حزمة SYN → إذا رد الهدف بـ SYN/ACK → المنفذ مفتوح. إذا لم يرد → غالباً firewall blocking.

### **ACK Scan (-sA)**

يرسل حزمة ACK فقط → إذا رد الهدف بـ RST → المنفذ غير مصفى (unfiltered). إذا لم يرد → الحزم تُسقط (filtered).  
**ميزة مهمة:** معظم الجدران النارية لا تمنع ACK لأنها لا تحدد هل الاتصال بدأ داخلياً أو خارجياً.

**مثال:**

`sudo nmap 10.129.2.28 -p 21,22,25 -sA -Pn -n`

- لو منفذ 22 يرد بـ RST → يعني ليس محمي (unfiltered).
    
- لو 21 و 25 لا يردان → يعني الحزم تُسقط (filtered).
    

---

## **4. الكشف عن IDS/IPS**

- IDS/IPS قد يحظر IPك إذا عملت فحص عدواني (Aggressive Scan).
    
- إذا انقطع وصولك للهدف بعد الفحص → غالباً هناك IPS.
    
- الحل: استخدام **عدة VPS أو تغيير IPك**.
    

---

## **5. تقنيات التخفي وتجاوز الحماية**

### **أ. استخدام Decoys (-D)**

- يضيف Nmap عناوين IP وهمية ضمن الحزم لإخفاء عنوانك الحقيقي.
    
- مثال:
    

`sudo nmap 10.129.2.28 -p 80 -sS -Pn -n -D RND:5`

يعني استخدم 5 عناوين عشوائية كـ Decoys.

- **تحذير:** يجب أن تكون الـ Decoys حقيقية (حية) وإلا قد تسبب SYN-flood.
    

---

### **ب. تغيير عنوان المصدر (-S)**

- تختبر هل الجدار يمنع شبكات محددة فقط.
    

`sudo nmap 10.129.2.28 -p 445 -O -S 10.129.2.200 -e tun0`

- لو المنفذ مفتوح بهذه الطريقة → يعني الحظر يعتمد على مصدر الاتصال.
    


---

### **ج. استخدام منفذ موثوق (--source-port 53)**

- الجدران النارية غالباً تسمح بحركة DNS (Port 53).
    
- إذا استخدمت منفذ 53 كمصدر للفحص، قد يمر بدون حظر:
    

`sudo nmap 10.129.2.28 -p 50000 -sS --source-port 53`

- إذا المنفذ صار open بدل filtered → يعني الحماية ضعيفة على هذا المنفذ.
    
- يمكنك حتى الاتصال مباشرة باستخدام netcat:
    

`ncat -nv --source-port 53 10.129.2.28 50000`

---

## **6. ملخص الأدوات والخيارات**

- **-sA** → ACK scan لتحديد القواعد.
    
- **-D RND:5** → استخدام Decoys لتخفي المصدر.
    
- **-S IP -e ifaceج** → تغيير عنوان المصدر وواجهة الإرسال.
    
- **--source-port 53** → تمرير الفحص من منفذ موثوق.
    
- **--dns-server IP** → استخدام DNS داخلي موثوق.
    
- **-Pn -n --disable-arp-ping** → منع اكتشافك عبر Ping و DNS.
    
- **--packet-trace** → متابعة كل حزمة لترى ردود الهدف.
    

---

## **الخلاصة**

- **Firewalls** يمكن اكتشاف قواعدها باستخدام ACK scan.
    
- **IDS/IPS** يمكن كشفها إذا حُظر IPك بعد الفحص.
    
- **Decoys, Source Port Spoofing, Changing IP** تساعدك على التخفي.
    
- **DNS port (53)** غالباً ثغرة لتجاوز الحماية.
    
- يجب أن تكون حذراً لأن زيادة العدوانية قد تؤدي لحظرك نهائياً.

# Lab Med

هنا المطلوب إنك تجيب **إصدار الـ DNS Server** اللي شغال على الهدف.  
ده بيتعمل عادة باستخدام **Nmap NSE script مخصص للـ DNS** بدل ما تعمل فحص شامل.

---

## أفضل طريقة 

لو عارف إن السيرفر بيشغل DNS (عادة على المنفذ 53)، نفذ:

`sudo nmap -sU -p53 --script=dns-nsid <TARGET_IP>`

- `-sU` → فحص UDP (لأن DNS غالباً شغال UDP).
    
- `-p53` → المنفذ بتاع DNS.
    
- `--script=dns-nsid` → سكريبت Nmap بيطلب من الـ DNS السيرفر يبعته اسمه وإصداره (NSID).
    

---

## طريقة بديلة لو TCP متاح

لو DNS بيشتغل كمان على TCP (مش شائع لكن ممكن)، تستخدم:

`sudo nmap -sV -p53 <TARGET_IP>`

ده هيحاول يجمع إصدار السيرفر عن طريق الـ Banner.

---

## في أسوأ الأحوال (لو ما ظهرش بانر)

ممكن تستخدم أدوات زي:

`dig version.bind txt chaos @<TARGET_IP>`

- `dig` يسأل السيرفر مباشرة عن الـ version (لو الـ query مفتوح).
    
- الـ `CHAOS` class دي استعلام خاص بيجيب معلومات السيرفر لو مش مقفول.
